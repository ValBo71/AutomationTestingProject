## 🚀 CI/CD Pipeline (GitHub Actions)

This repository includes a professional continuous integration (CI) pipeline implemented via GitHub Actions to validate the main test project automatically.

### Pipeline Overview
* **Target Project:** `01.Playwright/3.AutomationАtАutomationexercise.com/AutomationExercise.sln`
* **Technology Stack:** C#, .NET 9, Playwright for .NET, NUnit, and Allure Reports.
* **Trigger Events:**
  * Every `push` to the `main` branch.
  * Every `pull_request` targeting the `main` branch.
  * Scheduled run: **Every day at 07:00 AM Bulgarian time** (04:00 UTC).
  * Manual execution via the GitHub UI (`workflow_dispatch`).


### What the Pipeline Does
1. **Environment Setup:** Provisions a clean `ubuntu-latest` runner and installs the **.NET 9 SDK**.
2. **Restore & Build:** Restores NuGet packages and compiles the solution in `Release` mode.
3. **Playwright Installation:** Downloads required browser binaries (Chromium, Firefox, WebKit) along with their Linux system dependencies.
4. **Execution:** Runs the NUnit tests in **headless mode** to ensure optimal performance on CI.
5. **Artifact Archiving:** Generates and uploads the following test artifacts (available even if tests fail):
   * **TRX Test Results:** Raw XML test execution results.
   * **Failure Screenshots:** Full-page screenshots captured on test failures.
   * **Allure Results:** Raw XML/JSON files generated by Allure, ready for report generation.
   * **Traces & Logs:** Debug logs or Playwright trace files (if generated).

### QA Automation / SDET Portfolio Showcase
This setup demonstrates:
* **CI/CD Best Practices:** Infrastructure-as-code automation using YAML workflows.
* **Hermetic & Headless Execution:** Runtime configuration overrides (using environment variables) to separate local headed debugging from CI headless speed.
* **Failure Diagnostics:** Automated capture and persistence of system screenshots and test logs to easily triage CI build failures.
